# AI 驱动的竞品分析 Agent 协作系统 —— 核心目标与规划

> **项目愿景**：打造一个模拟真实数字调研小组的多 Agent 协作系统，将传统重人力、重经验的市场洞察过程，转化为高度自动化、结构化且完全可追溯的 AI 工业流水线。

---

## 1. 项目背景与业务痛点

* **流程极度繁琐**：传统竞品分析需经历大量重复性的人工搜索、复制与排版作业。
* **信息源极度分散**：公开市场数据散落各处，收集、比对与清洗的综合成本极高。
* **重度依赖个人经验**：分析深度与客观性受限于执行者的个人行业认知，团队难以沉淀标准化的调研方法论。

## 2. 核心系统机制

* **DAG 任务工作流**：构建有向无环图（DAG）式的任务调度引擎，确保各环节按严格的依赖关系有序流转。
* **Schema 驱动协作**：摒弃松散文本传递，各环节基于自定义的“竞品知识 Schema”进行强类型的数据沉淀与状态通信。
* **交叉审查与反馈闭环**：打破单向执行链路，引入独立的质检机制，通过审查反馈触发上游节点的动态修正。

## 3. 核心 Agent 角色划分

| Agent 角色 | 核心职责 | 上游依赖 | 输出产物 |
|---|---|---|---|
| **采集 Agent** | 自动搜索并抓取目标竞品在公开渠道的信息源。 | 无 | 清洗后的网页原文与采集元数据 |
| **抽取 Agent** | 将非结构化的原始网页信息，抽取为符合特定 Schema 的属性。 | 采集 Agent | 结构化竞品知识 (JSON) |
| **分析 Agent** | 进行多维度的商业横向对比、SWOT 推演与趋势洞察。 | 抽取 Agent | 深度商业分析结论 |
| **撰写 Agent** | 消费分析结论与原始证据，组装高可读性的商业研究报告。 | 分析 Agent | 结构化排版的长文报告 |
| **质检 Agent** | 审查报告的逻辑自洽性、结构完整性与内容客观性。 | 撰写 Agent | 质检评分与明确的修订建议列表 |

## 4. 关键工程约束（非功能性要求）

* **结果绝对可溯源**：杜绝大模型幻觉，强制要求每一条核心分析结论必须关联具体的证据源（如来源 URL、网页标题或原文片段），做到条条结论有据可查。
* **系统高度可观测**：彻底打破 AI 执行的“黑盒”，完整持久化每个 Agent 的决策流、执行计划、输入输出片段、以及状态流转轨迹，确保全过程完全透明。

# AI 竞品分析 Agent 协作系统 - 开发规范

## 1. 核心编码铁律 (必须遵守)

- **中文注释:** 所有业务逻辑、核心方法、复杂的条件判断，必须加上详细的中文注释。在解释代码时，要把对应的代码块也粘贴出来。
- **面向对象:** 严格遵循单一职责原则，Agent 的职责必须明确拆分（采集、分析、撰写、质检）。
- **容错机制:** 所有调用外部大模型 API 或外部抓取工具的地方，必须有 try-catch 异常处理和重试机制（Max Retries）。
- **无幻觉原则:** 数据 Schema 中必须强制包含 `sourceUrls` 字段，确保分析结果可溯源。
- **任务计划与进度可视化:** Agent 执行采集、分析、撰写、质检任一核心任务前，必须生成结构化执行计划，
- 计划需包含「任务拆解步骤、各步骤核心目标、预期耗时、依赖前置条件」；
- 任务执行过程中需实时输出进度信息，明确展示「当前执行步骤、已完成步骤占比、剩余步骤、步骤执行状态（成功/失败/待执行）」；
- 计划和进度需采用统一结构化格式（如 Markdown 列表/JSON），便于系统解析和用户查看，且进度信息需持久化记录，支持任务中断后恢复时追溯当前进度。
- **Agent 运行时输出规范（Prompt 层要求，非代码实现）：** 各 Agent 在执行任务时，必须在 LLM 输出中严格按以下格式报告当前状态。此条为 Agent prompt 模板的输出格式约束，由 PromptTemplateService 注入到各 Agent 的 system prompt 中，不属于后端代码逻辑实现范畴：
- 当前阶段：[正在进行的阶段]
- [x] 信息采集：已完成
- [ ] 数据分析：执行中
- [ ] 报告撰写：待执行

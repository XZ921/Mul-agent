# Phase 4 Task RAG 与对话边界

## 当前定位

Phase 4 的 `Task RAG` 只提供围绕单个任务运行态、证据链和恢复动作的解释能力。这个阶段的能力定位是 **MVP**，重点是把任务详情页、恢复动作和解释入口统一起来，而不是提前承诺完整智能助理。

## 解释型统一入口

- 对话入口是一个**解释型统一入口**，负责回答“当前为什么停住”“缺哪些证据”“恢复后会做什么”。
- 该入口优先读取任务快照、节点状态、报告诊断和检索证据，不直接替代后端编排器执行复杂业务决策。
- 当用户提出执行意图时，对话层先生成动作预览，再由任务命令入口完成真正的执行。

## Task RAG 的 MVP 范围

- `Task RAG` 可以检索当前任务沉淀的证据、知识片段、报告诊断和计划版本信息。
- `Task RAG` 可以把这些证据组织成可解释回答，帮助用户理解阻塞原因、补证方向和恢复影响。
- `Task RAG` 不负责跨任务记忆融合，不负责泛化成全局知识问答，也不负责替代人工做高风险动作审批。

## 与 Phase 5 的明确边界

- `Phase 5` 才会承接更强的跨任务协同能力，本阶段不能提前宣称已经具备**完整记忆融合**。
- `Phase 5` 才会逐步引入更复杂的策略编排、组织级知识复用和更强的权限控制，本阶段不承担**复杂动作接管**。
- 因此，Phase 4 文档、接口和前端提示都必须明确说明：当前是围绕任务详情页的解释与恢复闭环，而不是通用 AI 工作台。

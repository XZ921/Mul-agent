# 05 Knowledge Ingestion RAG And Memory

## 文档目的

本文档用于定义系统如何从多源资料入口出发，形成统一的知识摄取、RAG 召回与记忆沉淀能力。

本文档当前定位为“知识系统总览文档”，统一覆盖资料摄取、RAG 和记忆三层能力；后续不要求与其他文档保持相近篇幅，可以自然扩展。

## 本能力域在系统中的职责

本能力域负责：

1. 定义资料如何进入系统。
2. 定义资料如何被清洗、切片、索引、召回和回指。
3. 定义任务记忆、领域记忆和组织记忆的长期形态。

## 业务目标与用户价值

1. 让用户不必一次性准备完整资料。
2. 让系统可以在任务中持续补证和复用知识。
3. 让后续任务能够继承前序积累，而不是每次从零开始。

## 当前结论

最终知识来源统一分为四类：

1. `User Provided`
2. `AI Discovered`
3. `Uploaded Documents`
4. `Authenticated Sources`

最终知识链路应为：

`搜索 / 上传 / 连接器接入 -> 清洗解析 -> 切片索引 -> 检索召回 -> 重排筛选 -> 上下文装配 -> Agent 使用`

本文档后续允许自然膨胀到更高细节密度；当“资料摄取”“RAG 检索”“记忆体系”中任一子域出现独立接口体系、独立路线图或独立约束集合时，再考虑拆分子文档。

## 目标能力

1. 多源资料统一接入
2. 统一的文档解析与切片标准
3. 任务级 RAG、领域级 RAG、组织级 RAG
4. 可审计的召回与证据回指
5. 短期记忆与长期记忆协同

## 关键场景

1. 用户只提供少量网址，系统主动补源并形成任务知识底座。
2. 用户补传 PDF 或内部导出资料，系统纳入后续召回链路。
3. 由于某章节缺证据，系统定向补搜并更新任务级索引。
4. 历史相似任务的模板、术语和资料被复用到新任务。

## 核心对象 / 核心模块

1. `Source Discovery Layer`
2. `Document Parsing Layer`
3. `Chunking & Indexing Layer`
4. `Retrieval & Rerank Layer`
5. `Context Assembly Layer`
6. `Task Memory`
7. `Domain Memory`
8. `Organization Memory`

## 核心流程 / 状态流 / 编排关系

### 资料摄取流程

1. 系统接收用户资料、AI 发现结果或连接器数据。
2. 对原始资料做清洗、结构化解析与元数据提取。
3. 生成 `KnowledgeDocument` 与 `RetrievalChunk`。
4. 写入对应层级的检索索引。
5. 在任务节点执行时，根据当前上下文做召回与重排。
6. 最终将被采用的内容回指到证据与来源对象。

### 记忆分层

1. `短期记忆`
   当前任务运行态现场、阶段结论和临时上下文。
2. `任务记忆`
   单任务的历史执行、修订和证据沉淀。
3. `领域记忆`
   跨任务复用的术语、模板、行业知识。
4. `经验记忆`
   对搜索词、来源质量、修订规律等经验的长期沉淀。

## 关键约束与设计原则

1. `RAG 不是 Evidence 的替代品`
   RAG 解决召回问题，不取代事实载体。
2. `知识来源必须分类`
   不同来源在可信度、权限和处理策略上必须可区分。
3. `受限资料必须有边界`
   登录受限资料不能被默认当作公开网页处理。
4. `记忆不等于向量库`
   向量检索只是记忆体系的一部分。

## 扩展预留

1. 后续可扩展飞书、Confluence、Notion 等连接器。
2. 后续可引入图片、音视频和 OCR 等多模态资料接入。
3. 后续可支持组织级知识治理与资料生命周期管理。
4. 若知识系统规模明显扩大，可进一步拆出“摄取与解析”“检索与重排”“记忆与知识治理”独立 spec。

## 待确认问题

1. 任务级 RAG 与领域级 RAG 的隔离策略应在何时正式固定。
2. 长期是否需要把经验记忆单独提升为推荐或优化引擎。

## 本阶段不约束的内容

1. 不强制要求当前就选定唯一向量数据库实现。
2. 不强制要求所有资料类型都在当前阶段上线。

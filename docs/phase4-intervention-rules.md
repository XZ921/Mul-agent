# 第四阶段人工干预规则收口

本文档用于收口 V2 第四阶段的人工干预语义，要求与当前代码保持一致。

## 1. 当前已支持的人工干预能力

### 1.1 任务级

1. `resumeTask`
   适用状态：`FAILED`、`STOPPED`
   语义：尽量保留成功节点成果，只重置未完成、失败、跳过或中断的节点。

2. `retryTask`
   适用状态：`FAILED`
   语义：整任务重置，删除报告、知识、证据、Agent 日志等派生产物后重新执行。

3. `stopTask`
   适用状态：`RUNNING`
   语义：把任务标记为 `STOPPED`，并把尚未完成的节点统一标记为 `SKIPPED`。

4. `executeTask`
   当前前端入口仅对 `PENDING`、`FAILED` 开放。
   语义：从任务维度重新启动执行，不属于节点级人工干预主入口。

### 1.2 节点级

1. `rerunFromNode`
   适用状态：任务不处于 `RUNNING`
   语义：从指定节点开始，自动识别当前节点及其受影响下游节点，统一失效并重跑。

2. `updateNodeConfigAndRerun`
   适用状态：任务不处于 `RUNNING`，且节点存在 `nodeConfig`
   语义：先更新节点配置，再按 `rerunFromNode` 规则识别受影响分支并继续执行。

3. `pauseNode`
   适用状态：节点为 `PENDING`
   语义：把单节点切换为 `PAUSED`，该节点不会继续参与当前 DAG 调度；恢复前，下游保持等待。

4. `resumeNode`
   适用状态：节点为 `PAUSED`
   语义：把单节点恢复为 `PENDING`；若任务已因暂停节点收口，则自动继续执行。

5. `skipNode`
   适用状态：节点为 `PENDING` 或 `PAUSED`
   语义：把单节点直接标记为 `SKIPPED`，并按既有依赖规则处理下游节点。

6. `terminateNode`
   适用状态：节点为 `PENDING`、`PAUSED` 或 `RUNNING`
   语义：
   - `PENDING` / `PAUSED`：立即标记为 `SKIPPED`
   - `RUNNING`：写入协作式终止请求，不做线程级强杀；待当前轮执行返回后丢弃结果并标记为 `SKIPPED`

7. 搜索检查点复用
   适用节点：`COLLECTOR`
   语义：若节点输出中存在 `searchAudit`，则在 `resume` 或 `rerun` 前回写 `searchAuditCheckpoint` 到节点配置，优先复用候选来源与选源现场。

## 2. 下游失效策略

1. 影响范围包含“当前节点 + 所有直接/间接下游节点”。
2. 未受影响的成功节点保持不变，不会被误清空。
3. 采集节点重跑时，只删除该采集分支对应的证据前缀数据。
4. 若影响范围覆盖 `extract_schema`，则删除结构化知识结果。
5. 若影响范围覆盖 `analyze_competitors`、`write_report`、`rewrite_report`、`quality_check`、`quality_check_final` 等报告链路节点，则按分支影响删除对应报告与质量产物。

## 3. 恢复与重跑的差异

1. `resumeTask`
   目标：修复中断或失败链路。
   保留策略：所有已成功节点尽量保留。

2. `rerunFromNode`
   目标：从用户指定节点重新计算某一分支。
   保留策略：仅保留未受影响分支的成功节点成果。

3. `updateNodeConfigAndRerun`
   目标：人工修改节点策略后继续执行。
   保留策略：与 `rerunFromNode` 一致，但会以新的 `nodeConfig` 作为起点。

## 4. 当前仍受限但已明确边界的能力

1. 运行中节点的“终止”是协作式终止请求，不保证立即中断底层 LLM / 浏览器调用。
2. 不支持线程级、进程级或外部资源级的硬中断。
3. 单节点暂停当前只对尚未启动的 `PENDING` 节点开放，不提供运行中节点就地挂起。
4. 不支持在任务 `RUNNING` 时直接修改节点配置。

这些边界需要在前端说明文案和接口返回中保持一致，避免误导为“立即强杀”。

## 5. 前后端对齐要求

1. 任务详情页按钮必须以后端返回的可操作能力字段为准，不能只靠前端硬编码状态猜测。
2. 节点追踪抽屉必须展示：
   - 是否允许从该节点重跑
   - 是否允许修改配置后继续
   - 是否允许暂停、恢复、跳过、终止
   - 本次重跑会影响的节点范围
   - 是否存在可复用检查点
3. 若后续修改 `resume`、`rerun`、`stop`、检查点复用等语义，必须同步更新本文档。
